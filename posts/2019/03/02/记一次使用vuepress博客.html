# 记一次使用Vuepress博客踩的坑

使用的是meteorlxy博客主题

主题官网是：官网

首先根据官网的方法

# 安装主题

创建一个新的项目my-blog：

mkdir my-blog
cd my-blog

安装vuepress状语从句：vuepress-theme-meteorlxy，添加注意next标签安装1.x版本：

npm install vuepress@next vuepress-theme-meteorlxy@next

注意：

现在Vuepress 1.x还处于Alpha阶段，经常会发生改动，如果直接安装@next后使用主题出现问题的话，建议按照本主题的peerDependencies要求锁定对应的vuepress和相关包的版本号。例如：

npm install \
  vuepress@1.0.0-alpha.35 \
  @vuepress/core@1.0.0-alpha.35 \
  @vuepress/markdown@1.0.0-alpha.35 \
  @vuepress/markdown-loader@1.0.0-alpha.35 \
  vuepress-theme-meteorlxy@1.0.0-alpha.13

是的，你必须手动锁定所有相关包的版本号，因为Vuepress并没有锁定它们（这确实很不好）

其中 我是使用全局安装的 Vuepress 在这过程中还是有一些坑：

使用yarn 或者 cnpm 强制版本安装(就像上面注意写到的那样，由于目前是alpha版本必须要安装指定版本本才可以，正确使用)不会按照强制安装的版本来安装，比如主题作者提到的vuepress@1.0.0-alpha.35 以及 @vuepress/markdown@1.0.0-alpha.35 等。

即使使用强制版本安装 还是会 装上 当前时间最新版本 vuepress@1.0.0-alpha.40 这会出问题的。

目前我提供一种方法：

这个issues里有提到https://github.com/Bloss/vuepress-theme-yubisaki-example/issues/1

workaround：

"vuepress": "1.0.0-alpha.32",
"@vuepress/core": "1.0.0-alpha.32"
在项目根目录下更改并添加这两个版本，安装依赖之后:

cd node_modules/vuepress, rm -rf node_modules, 并且将 vuepress 的依赖改为 @vuepress/core: 1.0.0-alpha.32，在 vuepress 目录下执行 npm install 重新安装依赖。

回到项目根目录，npm run dev，跑起来了

感谢回答，我的解决方法是：
全局安装vuepress指定版本
cnpm i vuepress@1.0.0-alpha.35 -g //这个不会直接安装指定版本还是会安装 1.0.0-alpha.40版本
所以只能去全局安装根目录 我的全局安装目录是：C:\Users\小刘\AppData\Roaming\npm\node_modules\vuepress
然后按照您的方法rm -rf node_modules
接着手动安装每一个依赖
(cnpm install 还是会安装 1.0.0-alpha.40版本 所以只能曲线救国 手动安装指定版本了)
cnpm i @vuepress/core@1.0.0-alpha.35 
cnpm i @vuepress/theme-default@1.0.0-alpha.35
cnpm i @cac@6.3.9
最后按照您的方法
回到项目根目录，npm run dev，跑起来了
最后非常感谢您的回答！谢谢！

就如以上方法所示，我就是这样解决的。

另外我所在的时间点来说，一定要安装vuepress-theme-meteorlxy@1.0.0-alpha.13这个版本，我尝试安装其他版本会报错~
